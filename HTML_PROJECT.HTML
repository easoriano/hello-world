Introduction

Coined by Alex Russell and Frances Berriman Progressive Web Apps are not a new framework or technology. It's a practice which looks to make a web application function similar to a native desktop or mobile application. The goal is to create an experience so that the user is unable to tell the difference between a Progressive Web App and a native desktop or mobile app.

Progressive web applications or PWA, work by using progressive enhancement. Which means that a PWA will perform the same functions on a new desktop or mobile device as it would on an older version.

PWAs feel and behave like native apps. They sit in a user’s home screen, send push notifications like native apps, and have access to a device’s functionalities like native apps this provides an experience that is consistently fast. Because user's can cache the data from the moment they start interacting with PWA's, everything happens really fast, even without hitting the network. This is done with the help of service workers. It can help paint a picture on a user’s.

Skills you should have before implementing

TThe tools to require so as to implement or build PWAs are provided by Google. Using, Google's guide to building your first progressive web app. There you will find a checklist of items for progressive web apps. This checklist takes into account that the developer has knowledge of JavaScript, CSS, HTML and JSON files. The following is the checklist.

  • Site is served over HTTPS.
  • Pages are responsive on tablets and mobile devices.
  • All app URLs load while offline.
  • Metadata provided for add to Home screen
  • First load fast even on 3G.
  • Site works across browsers.
  • Page transitions don't feel like they block on the network.
  • Each page has a URL.
    • Tools and items needed to implement

    There are 4 main items required to develop a PWA. The first is a Web App Manifest, this is a file that gives you information such as, the icon of the app, background color of the app, name of the app, short name, and so on. We can write this manifest file ourselves or we can use tools to generate one for us.

    Next you'll need Service Workers. These are event-driven workers which are JavaScript files that run in the background of the application, separately from the main browser thread. They intercept network request, caching or retrieving resources from the cache, and delivering push messages. They can be used to load data for offline use.

    The third item required is an Icon. The manifest tool stated above assist in generating icons. However, if a you are looking to create your own Icon any jpeg image will work.

    The final Item required is for the web application to work over a secure network, HTTPS. This is required for all PWAs, it helps in preventing middle man attacks. It also establishes the application as a trusted and reliable site. Securing a site is a good practice and services like Cloudfare and LetsEncrypt, make it easy to get the required SSL certificate.

    The Pro's and Con's

    Progressive Web Apps appear to be perfect with their functionality and speed, however as with any technology there exist positives and flaws.

    THE PRO'S:

  • PWAs are a combination of native and web apps, they have the feature of being accessible online, just like native apps.
  • It can be used when there is limited or no connectivity. This is because the data is cached upon its last interaction with the internet thus, PWAs are still accessible.
  • Due to the technology used to build PWAs, such as, app shell, web manifest, and service workers. PWAs maintain the speed of a native app.
  • PWAs download quickly on to mobile devices or desktops. Users are not directed to the app store when looking to install PWAs.
  • Unlike native apps, PWAs are hosted on the web server. This mean that storage space on mobile devices or desktops is small. On average around 2MB of space is used.
  • The use of push notifications are a vital part of the native application. Thanks to the use of Service Workers PWAs can push notifications as well.

    • THE CON'S

  • The benefit of PWAs do not need to rely on the app stores for downloads is not as great as it sounds. It provides for quick downloads, however the app store is where many users go to search for apps. Thus, it misses out on user traffic.
  • PWAs work just as native app do, but they are not native apps, and so some restrictions do exist. There are feature that can be accessed by native apps that cannot be accessed by PWAs. Features such as, Bluetooth, proximity sensors, advanced camera controls, geofencing, contacts and many more.
  • PWAs may be a combination of native apps and web apps but this combinations comes with a cost to the battery. PWAs use more battery than most native apps. This may cause users to use a PWA only when battery life is not important.

    • There are clear advantages and disadvantages of using PWAs, but it is clear that the advantages outweigh the disadvantages. No technology is perfect, and as users use mobile devices more often than desktops for their online activity. PWAs help bridge the gap between native apps and web apps.

    Implementation

    To implement the code required to build a PWA one should have an understanding of JavaScript, CSS, HTML, and JSON files. You should also be able to create, move, and keep track of multiple files.

    Start with and index HTML file.

    <!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>Page Title!</title> <link rel="stylesheet" href="/styles.css"> </head> <body> <header> <h1 class="center">Heading</h1> </header> <main> <div class="container"></div> </main> <script src="app.js"></script> </body> </html>

    The above code has a link to styles.css this takes into account the fact that you have created an external style sheet. If this is not the case then the link should be removed.

    Next add the ability to fetch by adding the following to your initial HTML file.

    <script> async function fetch() { const res = await fetch(`https://`); const json = await res.json(); main.innerHTML = json.data.map().join('\n'); } </script>

    You can create your own Manifest.json file using a code similar to the following manifest;

    { "name": "Name", "short_name": "Name", "icons": [{ "src": "images.png", "sizes": "128x128", "type": "image/png" }], "start_url": "/index.html", "display": "standalone", "background_color": "#3E4EB8", "theme_color": "#2F3BA2" }

    Now you'll have to create a service worker, this is done using JavaScript here is an example of a service worker that could be used after updating the required information so as it fits your PWA. This example is for a PWA on meme's.

    const staticAssets = [ './', './styles.css', './app.js' ]; self.addEventListener('install', async event => { const cache = await caches.open('static-meme'); cache.addAll(staticAssets); }); self.addEventListener('fetch', event => { const {request} = event; const url = new URL(request.url); if(url.origin === location.origin) { event.respondWith(cacheData(request)); } else { event.respondWith(networkFirst(request)); } }); async function cacheData(request) { const cachedResponse = await caches.match(request); return cachedResponse || fetch(request); } async function networkFirst(request) { const cache = await caches.open('dynamic-meme'); try { const response = await fetch(request); cache.put(request, response.clone()); return response; } catch (error){ return await cache.match(request); } }

    This service worker will help us cache data and fetch resources. You will have to register this service worker in your HTML file. Here is an example of code you can use to do just that.
